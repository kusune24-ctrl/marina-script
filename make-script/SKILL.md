# make-script スキル — データドリブンまりなYouTube台本作成

過去の投稿データ（視聴維持率・視聴者コメント・競合の企画パワー）を分析し、それを根拠に企画を決め、YouTubeノウハウ体系に沿った台本を生成する。テーマ指定があればそれを軸に、無ければデータからベスト企画を提案する。

### ② 分析（3データから勝ち筋と弱点を抽出）

生成された JSON を Read し、最低限この3点を言語化する：
- **勝ちフォーマット**：まりな過去ヒット＋競合の企画パワー上位の共通項（型・尺・選の数・切り口）
- **視聴者の欲求とブロッカー**：コメントから「最も求めている手段」「諦め・反発」を抽出（例：「金が有るか無いか」＝年収顔への諦め）
- **維持率の弱点**：`yt_retention.json` から（実測知見：**冒頭30秒で25〜30%離脱／5選×20分が後半維持・CTA到達で最良／9選×26分は最悪**）

### ③ 企画決定（YouTubeノウハウ体系に準拠）

メソッドは `outputs/loom_transcripts_20260615/`（企画編・台本編16本）。要点：
- **企画＝切り口**。欲求(ニーズ＝再生上限のパイ)と手段(達成方法)を分けて考える。視聴者が最も欲しい手段を当てる
- **タイトル**：【スミカッコ】＝サムネで補えない最強訴求ワード／後半＝ターゲット明示・SEO。競合が使う実証ワードを使う
- **サムネ**：欲求の解消＝手段を直感で見せる
- 企画パワー6倍以上の競合ヒットから再現可能な型を選ぶ

→ ベスト企画（タイトル3案＋サムネ案）をデータ根拠つきで提示。

### ④ 台本作成（メソッド＋維持率を反映）

下記「まりなキャラ設定」「台本フォーマット」で執筆。ノウハウ体系の技術を必ず実装：
- **冒頭は30秒勝負**：最初の10秒で常識破壊＋権威（最大流出点の対策）。1分以内に期待値・新規性を提示
- **前提のすり合わせ**を冒頭直後に短く入れる
- **PREP＋超具体エピソード**（時間・場所・年齢・セリフまで描写。元キャバ嬢/指導生の実話）
- **接続詞はウインカー**（なんでかって言うと/つまり/逆に/だから）
- **130字に1回は視聴者へ問いかけ**（離脱防止）
- **噛み砕き**（小学生でも分かる言葉・二字熟語は言い換え・面白い例えで緊張と緩和）
- **エンディングはヌルッと**（「ということで最後に」等の終了宣言をしない）
- **CTAは意図を透けさせない**（メッセージ先行＝「教材より明日1つやる方が本体」、LINEはサラッと最後に）
- **尺は18〜20分（セリフ約6,000〜7,000字）／項目は5選**を基本（維持率データ準拠）

## 台本フォーマット

テーブル・カット番号・時間・指示欄は一切使わない。セリフをそのままテキストで書く。1文ごとに改行する。

```
# タイトル（【スミカッコ訴求】本題【ターゲット】）

## タイトル案（3）
## サムネ案

## 台本

【冒頭｜最初の10秒で常識破壊＋権威】
（最初の一文から強い断言。悩み言い当て→常識破壊→権威→宣言→ベネフィット）

【前提のすり合わせ】
（短く握る）

【特徴①〇〇】
（結論→なんでかって言うと→超具体エピソード→問いかけ→噛み砕き→命令で締め）

…②〜⑤…

【まとめ】
（5つを噛み砕いて再確認＋共通項）

【CTA】
（メッセージ先行→frontiaをサラッと→「本体は明日1つやること」）
```
